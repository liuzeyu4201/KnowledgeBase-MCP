# 知识库可视化模块开发计划

## 1. 模块目标

开发一个面向知识库场景的可视化网页系统，作为现有 MCP Server 的上层展示与操作入口。网页系统不直接访问数据库或 Milvus，所有分类、文档、任务状态等核心数据均通过后端调用 MCP Server 获取，保证网页层与 MCP 层的数据和状态一致。

## 2. 首期范围

### 2.1 已纳入本期

- 首页分类展示
- 分类下文档列表展示
- 可视化后端聚合 API
- 基于 WebSocket 的批量导入任务状态推送
- Nginx 反向代理接入
- 请求追踪、访问日志、业务日志、错误日志

### 2.2 暂不纳入本期

- 前端直接发起文档导入或删除操作
- 文档详情页、切片页、检索结果页
- 登录鉴权与权限控制
- 审计页、任务中心页、实时日志面板完整实现

## 3. 实施原则

- 网页层只做展示与交互，不绕过 MCP Server 拼装平行逻辑
- 页面数据必须来源于 MCP Tool 或由其结果做轻量聚合
- 所有实时状态都以 MCP 对外暴露的任务状态为准
- 页面行为与上传接口、批量任务、文档状态保持最终一致

## 4. 开发阶段拆分

### 第一阶段：页面基础骨架

- 建立前端静态资源目录
- 提供首页和分类页
- 确定统一视觉风格和响应式布局

### 第二阶段：后端聚合接口

- 封装网页后端到 MCP 的统一调用网关
- 提供分类聚合接口
- 提供分类文档列表接口
- 提供批量任务详情接口

### 第三阶段：实时能力

- 提供任务状态 WebSocket
- 实现任务终态自动结束推送
- 为后续实时日志流预留扩展点

### 第四阶段：部署与治理

- 补充 Nginx 代理配置
- 统一请求 ID / trace ID
- 完善访问日志、业务日志、错误日志

### 第五阶段：测试与文档

- 补充网页模块接口测试
- 补充 WebSocket 行为测试
- 更新测试报告

## 5. 当前实现结果

当前已经完成以下交付：

- `GET /` 和 `GET /ui`：知识库分类首页
- `GET /ui/categories/{category_id}`：分类文档列表页
- `GET /api/visualization/categories`：分类聚合接口
- `GET /api/visualization/categories/{category_id}/documents`：分类文档接口
- `GET /api/visualization/import-tasks/{task_id}`：任务详情接口
- `GET /ws/import-tasks/{task_id}`：任务状态 WebSocket
- `deploy/nginx/default.conf`：Nginx 代理配置

## 6. 后续迭代方向

- 新增上传入口页面，整合暂存文件上传与文档导入
- 新增搜索页，接入 `kb_search_retrieve`

- 新增异常中心与日志面板
