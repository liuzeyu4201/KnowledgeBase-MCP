# 知识库可视化模块设计说明

## 1. 模块定位

知识库可视化模块是现有 MCP Server 的上层网页系统，负责把分类、文档、批量导入任务等能力以网页方式呈现给用户。该模块不直接查询 PostgreSQL 或 Milvus，而是由 FastAPI 后端统一调用 MCP Tool，再把结果提供给前端页面或 WebSocket。

## 2. 技术栈

- 前端：HTML、CSS、JavaScript
- 后端：FastAPI
- 实时通信：WebSocket
- 代理：Nginx

## 3. 目录结构

```text
knowledgebase/
  web/
    mcp_gateway.py          网页后端访问 MCP 的统一网关
    middleware.py           请求追踪与访问日志中间件
    routes.py               页面路由、聚合 API、WebSocket
    static/
      css/app.css           样式

      pages/

  nginx/

```

## 4. 页面与路由设计

### 4.1 页面路由

- `GET /`
- `GET /ui`

说明：
- 返回知识库首页
- 展示分类卡片和分类文档数量

- `GET /ui/categories/{category_id}`

说明：
- 返回分类文档列表页
- 展示当前分类下的文档基础信息
- 支持通过 `?task_id=` 打开任务观察面板

### 4.2 后端聚合 API

- `GET /api/visualization/categories`

返回：
- 分类基础信息
- 文档总数
- 最近更新时间

- `GET /api/visualization/categories/{category_id}/documents`

返回：
- 分类信息
- 文档列表
- 分页信息

- `GET /api/visualization/import-tasks/{task_id}`

返回：
- 任务详情
- 子项列表

### 4.3 WebSocket

- `GET /ws/import-tasks/{task_id}`

行为：
- 后端轮询 `kb_document_import_batch_get`
- 将任务状态实时推送给前端
- 当任务进入终态后结束推送

## 5. 与 MCP Server 的集成方案

### 5.1 设计原则

- 网页模块不能绕过 MCP 直接查业务表
- 网页模块必须基于 MCP Tool 暴露的结果工作
- 网页层只允许做轻量聚合，例如补文档数量、补页面展示字段

### 5.2 调用链路

#### 首页分类页

1. FastAPI 后端调用 `kb_category_list`
2. 对每个分类调用 `kb_document_list`
3. 取分页总数作为 `document_count`
4. 聚合后返回前端

#### 分类文档页

1. FastAPI 后端调用 `kb_category_get`
2. FastAPI 后端调用 `kb_document_list`
3. 返回分类信息、文档列表和分页数据

#### 任务状态 WebSocket

1. WebSocket 建立连接
2. 后端循环调用 `kb_document_import_batch_get`
3. 把任务和子项状态推送给前端
4. 任务到达终态后停止推送

## 6. 一致性设计

### 6.1 数据一致性

- 前端只读 FastAPI 聚合接口结果
- 聚合接口只读 MCP Tool 返回结果
- MCP Tool 仍然是唯一业务入口

因此网页状态、后端状态、MCP 状态三者天然对齐。

### 6.2 错误处理

- MCP 调用失败时，网页 API 返回统一错误结构
- 页面不做本地伪状态维护
- WebSocket 出错时，向前端推送错误对象并断开连接

### 6.3 请求追踪

网页后端通过中间件注入：

- `X-Request-ID`
- `X-Trace-ID`

并把这两个值继续透传给 MCP Tool，用于后续链路排查。

## 7. 日志设计

### 7.1 访问日志

记录：

- 请求方法
- 路径
- 状态码
- 耗时
- request_id
- trace_id

### 7.2 业务日志

记录：

- 分类列表查询
- 文档列表查询
- 任务观察订阅

### 7.3 错误日志

记录：

- MCP 调用失败
- 页面聚合异常
- WebSocket 推送异常

## 8. Nginx 设计

Nginx 负责：

- 统一对外暴露网页入口
- 反向代理 FastAPI
- 支持 WebSocket 升级
- 统一限制上传体积

当前配置：

- `client_max_body_size 256m`
- `/mcp` 代理到 `app:8000`
- `/api/staged-files` 代理到 `app:8000`
- `/` 代理到 `app:8000`
- `/ws/` 开启 WebSocket 升级转发

访问边界约定：

- 本机调试允许直接访问 `127.0.0.1:8000`
- 公网访问必须统一经过 Nginx
- 因此宿主机上的应用与后端依赖默认只绑定本机回环地址，不直接暴露到公网
